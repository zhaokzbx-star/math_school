# 图像处理与结果保存规则

## 核心要求

### 1. 图片管理规则
- 代码生成的图片必须保存到对应的问题文件夹中（如 a1/, a2/, a3/, a4/）
- 新生成的图片必须覆盖替换同名旧图片
- 旧图片文件在新图片生成后自动被覆盖，无需手动删除
- **禁止在问题文件夹内创建同名子文件夹**（如：不允许在 a2/ 内创建 a2/ 子文件夹存放图片）

### 2. 结果文件规则
- 每个问题的运行结果必须保存为对应的 result 文件：
  - 第一问 → result1.md
  - 第二问 → result2.md
  - 第三问 → result3.md
  - 第四问 → result4.md
- 结果文件必须放置在对应的问题文件夹中
- 文件名必须严格按照 "result+数字" 的格式命名

### 3. 实时更新规则
- 每次运行代码后，必须立即更新图片和 result 文件
- 确保所有文件的时间戳是最新的
- 图片和结果必须与代码运行结果保持同步

### 4. 执行顺序
1. 运行代码生成图片（直接保存到问题文件夹根目录）
2. 根据运行结果更新 result 文件
3. 确认所有文件已正确保存

### 图片命名规范
- 图片命名格式：answer{问题号}图{序号}_{描述}.png
- 示例：answer4图一_帕累托前沿.png、answer4图二_成本效益曲线.png

### 强制执行
- 每次运行代码后必须检查并确认图片和 result 文件都已更新
- 不允许出现图片或结果文件过期的情况
- 不允许创建嵌套同名文件夹存放图片
