# 城市极端降雨下内涝韧性提升与疏散路径优化

<h1 align="center" style="font-size: 2em;">摘要</h1>

本文针对北方城市短时极端降雨引发的内涝灾害，以A市某3.2 km²中心城区片区一次历时1小时、累计雨量65 mm的暴雨事件为研究对象，围绕**积水动态演化、内涝韧性评价、动态疏散优化与韧性提升改造**四个核心问题，构建了“物理机理—系统评价—动态决策—工程优化”的完整数学模型体系，为城市内涝智慧治理提供定量化决策支撑。

**问题一**，基于水量平衡原理，构建引入管网排水折减系数$C\_d$与街区汇水乘数$\theta$的一维离散时间积水动态演化模型，刻画极端降雨下管网排水能力衰减与下垫面汇流放大效应；通过三次样条插值实现降雨序列逐分钟细化，并利用分段函数建立积水深度与通行能力系数、安全度的映射关系。采用最小二乘法反演标定参数，得到$C\_d = 0.100$、$\theta = 12.05$，模型拟合决定系数$R^2 = 0.947$，均方根误差仅5.631 cm，预测精度满足应急决策需求。结果显示，L5路段积水峰值达36.4 cm，60分钟时通行能力完全丧失，为片区内涝最高风险节点。

**问题二**，从抵御、恢复、适应三个维度构建包含9项指标的内涝韧性评价体系，提出MC-EWM-TOPSIS综合评价模型。通过蒙特卡洛模拟生成500组指标样本以量化输入不确定性，采用熵权法客观确定指标权重，测算得到研究片区现状综合韧性得分为0.4308，处于中等偏弱水平。敏感性分析表明，绿色调蓄容积（敏感度0.1978）是制约系统韧性的核心因素，预警覆盖率与抢修覆盖效率次之。基于加权TOPSIS的物理脆弱性指数识别出L5、L2、L8为高脆弱性路段，为靶向治理与疏散规避提供精准依据。

**问题三**，针对传统静态路径规划在动态积水场景下完全失效的痛点，建立以总疏散时间最短与路径安全度最高为双目标的动态时空规划模型。将时间维度嵌入路网拓扑，设计改进的时变Dijkstra算法，依据实时积水深度动态更新路段通行阻抗，并引入断路剪枝策略提升搜索效率。仿真结果表明，最优动态疏散方案路径为“起点→B→D→终点”，总耗时4.62分钟，综合安全度0.3240；传统静态最短路径因高风险路段断路，耗时14.25分钟，安全度仅0.0300。动态规划方案较静态导航疏散效率提升68%，安全度提升10.8倍，有效验证了本算法在规避内涝断路风险与保障疏散安全方面的显著优越性。

**问题四**，在5000万元预算约束下，以管网扩容长度、调蓄池数量、透水铺装面积与预警系统建设为决策变量，建立以总成本最低、韧性增益最大、退水时间最短为目标的多目标混合整数优化模型。通过蒙特卡洛采样生成15000个候选方案，采用帕累托支配关系提取非劣解集，并结合K-Means聚类归纳出三类典型改造方案：保守防御型（成本1226万元，韧性增益0.097，退水时间154.9 min）、均衡拐点型（成本2984万元，韧性增益0.171，退水时间125.5 min）与激进海绵型（成本4569万元，韧性增益0.265，退水时间106.9 min）。鲁棒性测试表明，在降雨强度增强15%、设施排水效能下降20%的复合极端扰动下，各方案退水延迟率均低于55%，方案具有良好的工程可靠性与场景适应性。

本文构建的“积水预测→韧性诊断→路径决策→改造优化”模型体系，实现了从状态刻画到工程决策的完整逻辑闭环，可为城市内涝实时风险监测、应急指挥调度及海绵城市基础设施改造提供科学的定量分析工具与决策支持。

## 二、问题分析

### 2.1 积水动态演化建模

本问题核心在于基于有限观测数据建立准确的积水深度预测模型，属于水动力学与参数估计范畴。通过水量平衡原理构建水动力学方程，利用实测数据标定模型参数，验证精度并分析误差来源。

### 2.2 内涝韧性综合评价

本问题旨在科学量化城市内涝韧性并识别薄弱环节，涉及多准则决策分析与敏感性分析方法。通过构建多层次评价指标体系，采用客观赋权法确定权重，计算综合韧性得分并定位薄弱环节。

### 2.3 动态疏散路径优化模型

本问题需在内涝动态变化场景下实现时间最短与安全度最高的双重目标，涉及动态规划与图论方法。通过构建时空网络模型，设计动态路径搜索算法，对比静态与动态方案的性能差异。

### 2.4 内涝韧性提升综合改造方案

本问题在有限预算约束下寻求改造措施的最优组合，实现成本、韧性、效率的综合最优，涉及多目标优化与帕累托最优理论。通过建立多目标优化模型，采用NSGA-II算法求解帕累托前沿，提取典型方案并进行鲁棒性分析。

## 三、模型假设

### 假设1：降雨均匀分布假设

研究区域内降雨强度时空分布均匀，严格遵循附表2给出的时段雨量与累计雨量时序数据，不考虑局部空间差异与降雨中心移动。

### 假设2：管网排水能力恒定假设

排水管网的设计排水能力严格取附表1给定值，运行期间保持恒定；管网超载后无压力流增排，溢流水量全部滞留于路段表面。

### 假设3：路面下渗率恒定假设

将不同路面类型（沥青、水泥）的下渗率视为常数，不考虑土壤含水量、压实度等因素的动态变化。

### 假设4：路段独立积水假设

各路段积水过程仅取决于该路段自身汇水面积、降雨入流及排水能力，周边地块径流在短历时内全部汇入相邻路段，忽略路段间的水流横向转移、漫溢及汇流延迟效应。

### 假设5：改造效果线性假设

各韧性提升措施对排水能力、调蓄容积、消退时间等指标的影响满足线性叠加，不考虑措施间的协同增益或边际递减效应。

## 五、模型的建立与求解

### 5.1 问题一：城市路网积水动态演化模型

城市内涝的本质是极端降雨下“汇水-排涝”物理过程的时空失衡。本问要求明确在 60 分钟极端暴雨历时内，8 条核心路段积水深度的动态演化过程。传统的三维水动力学漫溢模型（2D-SWE）计算量巨大且极度依赖精细的高程与管网拓扑数据，不适用于应急抢险阶段的快速推演。因此，本文基于水量平衡原理，引入“地形坡度放大因子”与基于标高落差的“管网顶托失效机制”，构建了一维离散时间积水动态演化模型\[5]。通过真实观测数据反演标定未知参数，实现了对内涝演变的高精度模拟与交通阻抗的科学映射。

#### 5.1.1 数据处理

原始降雨数据以5分钟为时间间隔给出，为获得连续时间尺度下的降雨输入，采用三次样条插值方法对降雨序列进行平滑处理，将其转换为逐分钟降雨强度$I\_{infiltration}$（m/min）。

#### 5.1.2 模型建立

考虑到复杂的二维或三维水动力模型计算耗时较长、基础数据要求高，往往不适用于应急防汛情况下的快速推演\[1]。因此，本文参考了学术界关于城市内涝多因素耦合的简化处理思路，基于地表水量平衡与一维管网排水机理，建立路段的积水深度动态演化方程\[1]：

$$
\frac{\partial h}{\partial t} = I\_{rain} - I\_{infiltration} - \frac{Q\_{drain}}{A} \tag{5-1}
$$

其中降雨强度$I\_{rain}$为单位时间单位汇水面积上的降雨体积（m/min），下渗率$I\_{infiltration}$为单位时间单位面积的下渗体积（m/min），$Q\_{drain}$为管网排水流量（m³/min），$A$为修正后的实际汇水面积（m²）。

排水流量$Q\_{drain}$由管网排水折减系数$C\_d$乘以设计排水能力$Q\_{design}$折减系数得到：
$$ Q\_{drain} = C\_d \cdot Q\_{design} \tag{5-2} $$
其中排水流量$Q\_{drain}$由管网排水折减系数$C\_d$乘以设计排水能力$Q\_{design}$折减系数得到：
$$ Q\_{drain} = C\_d \cdot Q\_{design} \tag{5-2} $$
汇水面积$A$由路段长度、宽度及汇水乘数确定：
$$ A = \theta \cdot L \cdot W \tag{5-3} $$

通行能力系数$C$与安全度$S$根据积水深度通过分段函数确定，反映积水对交通的不同影响程度：
$$
\text{$C$} =
\begin{cases}
1, & h < 5 \text{cm} \\
0.8, & 5 \leq h < 10 \text{cm} \\
0.4, & 10 \leq h < 20 \text{cm} \\
0.1, & 20 \leq h < 30 \text{cm} \\
0, & h \geq 30 \text{cm}
\end{cases} \tag{5-4}
$$

$$
\text{$S$} =
\begin{cases}
1, & h < 5 \text{cm} \\
0.9, & 5 \leq h < 10 \text{cm} \\
0.6, & 10 \leq h < 20 \text{cm} \\
0.2, & 20 \leq h < 30 \text{cm} \\
0, & h \geq 30 \text{cm}
\end{cases} \tag{5-5}
$$

#### 5.1.3 模型求解

采用有限差分法对式(5-1)进行离散求解：

$$
h\_{t+\Delta t} = h\_t + \Delta t \cdot \left(I\_{rain} - I\_{infiltration} - \frac{Q\_{drain}}{A}\right) \tag{5-6}
$$

参数标定采用最小二乘法，目标函数为预测值与实测值的均方误差：

$$
\min\_{C\_d, \theta} \sum\_{i=1}^{n} \sum\_{j=1}^{m} (h\_{ij}^{pred} - h\_{ij}^{obs})^2 \tag{5-7}
$$

其中 $h\_{ij}^{pred}$ 为第 $i$ 路段第 $j$ 时刻的预测积水深度，$h\_{ij}^{obs}$ 为对应实测值。

**参数约束**：管网排水折减系数$C\_d$反映了排水管网老化、堵塞等导致的实际排水能力折减，取值范围为$\[0.1, 1.0]$，其中1.0表示管网处于全新状态；街区汇水乘数$\theta$用于修正城市街区实际汇水面与路段投影面积的偏差，取值范围为$\[5.0, 40.0]$。
**参数约束**：

- $0.1 \leq C\_d \leq 1.0$（物理排水能力折减）；
- $5.0 \leq \theta \leq 40.0$（街区汇水乘数范围）。

#### 5.1.4 结果分析

**（一）参数标定结果分析**

本文采用序列最小二乘优化算法（SLSQP）对目标函数式(5-7)进行求解，标定得到管网排水折减系数$C\_d = 0.100$，街区汇水乘数$\theta = 12.05$。$C\_d$取值偏低（仅0.1）表明，在极端暴雨条件下，管网排水能力仅为设计值的10%，这与实际城市排水系统运行经验吻合——管网老化、淤积以及顶托效应的共同作用使得实际排水能力大幅折减。$\theta = 12.05$意味着街区实际汇水面积约为路段投影面积的12倍，说明城市地表汇流过程涉及大量周边用地（建筑屋顶、停车场、绿化等）的不透水汇水贡献。

### 5.2 问题二：内涝韧性综合评价与诊断模型

#### 5.2.1 解题思路与模型选择

本问题旨在建立内涝韧性综合评价模型，采用MC-EWM-TOPSIS算法进行系统层评价。MC（蒙特卡洛模拟）用于处理指标不确定性，EWM（熵权法）客观计算权重，TOPSIS基于理想解相似度计算综合得分；同时采用多准则路段脆弱性诊断方法识别薄弱路段。本问题从抵御、恢复、适应三个维度构建评价体系，基于问题一的积水数据为问题四的优化方向提供依据，评价结果用于验证方案有效性。

#### 5.2.2 数据处理

本问题数据来源于附表5，包含9个二级指标的现状值（Actual）、理想值（Ideal）与最差值（Worst），涵盖抵御能力、恢复能力、适应能力三个维度。每个指标包含方向属性（正向"+"表示越大越好，负向"-"表示越小越好），用于指导归一化方向。

预处理阶段采用截断正态分布对每个指标进行蒙特卡洛模拟，具体步骤如下：对第$i$个指标，以现状值$\text{Actual}\_i$为均值，以指标区间范围的15%作为标准差，即$\sigma\_i = 0.15 \times |\text{Ideal}\_i - \text{Worst}\_i|$，设定采样下界为$\text{low}\_i = \min(\text{Ideal}\_i, \text{Worst}\_i)$，上界为$\text{high}\_i = \max(\text{Ideal}\_i, \text{Worst}\_i)$，采用截断正态分布在这有限区间内进行500次随机采样，生成模拟矩阵$\mathbf{S} \in \mathbb{R}^{500 \times 9}$，确保所有采样值均落在物理合理范围内，从而充分考虑指标取值的不确定性。

#### 5.2.3 模型建立

**5.2.3.1系统层评价模型（MC-EWM-TOPSIS）**\[6]：

**步骤1-极向标准化**：对于正向指标（Direction="+"）：
$$x\_{ij} = \frac{v\_{ij} - {\text{Worst}}\_i}{{\text{Ideal}}\_i - {\text{Worst}}_i} \tag{5-1}$$
对于负向指标（Direction="-"）：
$$x_{ij} = \frac{{\text{Worst}}_i - v_{ij}}{{\text{Worst}}\_i - {\text{Ideal}}\_i} \tag{5-2}$$

其中$v\_{ij}$为第$i$指标第$j$次蒙特卡洛采样值，标准化后矩阵记为$\mathbf{X} \in \mathbb{R}^{500 \times 9}$。

**步骤2-熵权法计算权重**：
$$p\_{ij} = \frac{x\_{ij}}{\sum\_{j=1}^{500} x\_{ij}} \tag{5-3}$$
计算第$i$指标的信息熵：
$$e\_i = -\frac{1}{\ln 500} \sum\_{j=1}^{500} p\_{ij} \ln p\_{ij} \tag{5-4}$$
进而得到客观权重：
$$w\_i = \frac{1 - e\_i}{\sum\_{i=1}^{9}(1 - e\_i)} \tag{5-5}$$
\==**步骤2-熵权法计算权重**：为确保评价的客观性，本文采用 EWM-TOPSIS 评价模型，该方法能有效剔除主观打分偏差，精准反映指标间的真实差异\[2]。首先计算第 $i$ 指标的信息熵：
$$p\_{ij} = \frac{x\_{ij}}{\sum\_{j=1}^{500} x\_{ij}} \tag{5-6}$$
，再计算第$i$指标的信息熵：
$$e\_i = -\frac{1}{\ln 500} \sum\_{j=1}^{500} p\_{ij} \ln p\_{ij} \tag{5-7}$$
，进而得到客观权重：
$$w\_i = \frac{1 - e\_i}{\sum\_{i=1}^{9}(1 - e\_i)} \tag{5-8}$$

**步骤3-TOPSIS综合得分**：
$$d^+ = \sqrt{\sum\_{i=1}^{9} w\_i^2 (x\_i - 1)^2}, \quad d^- = \sqrt{\sum\_{i=1}^{9} w\_i^2 (x\_i - 0)^2} \tag{5-9}$$
综合韧性得分为相对贴近度：
$$C = \frac{d^-}{d^+ + d^-} \tag{5-10}$$

**指标分类**：

- **抵御能力**：路网连通度、排水能力达标率、低洼路段占比；
- **恢复能力**：积水消退半衰期、应急响应时间、抢修覆盖效率；
- **适应能力**：绿色调蓄容积、预警覆盖率、疏散点位密度。

**5.2.3.2路段层脆弱性诊断模型**：综合考虑路段五个维度的物理特性，采用Min-Max标准化后计算加权TOPSIS贴近度。设路段$r$的原始特征向量为$(\text{Elevation}\_r, \text{Drain\_Cap}\_r, \text{Slope}\_r, \text{Pipe\_Dia}\_r, \text{Max\_Depth}\_r)$，其中地形标高与坡度为正向指标（值越大越脆弱），排水能力与管径为负向指标（值越小越脆弱），最大积水深度为正向指标。

**步骤1-Min-Max标准化**：

$$
\begin{cases}
\text{Elevation}\_r' = \dfrac{\text{Elevation}\_r^{\max} - \text{Elevation}\_r}{\text{Elevation}\_r^{\max} - \text{Elevation}\_r^{\min}} \\
\text{Drain\_Cap}\_r' = \dfrac{\text{Drain\_Cap}\_r^{\max} - \text{Drain\_Cap}\_r}{\text{Drain\_Cap}\_r^{\max} - \text{Drain\_Cap}\_r^{\min}} \\
\text{Slope}\_r' = \dfrac{\text{Slope}\_r - \text{Slope}\_r^{\min}}{\text{Slope}\_r^{\max} - \text{Slope}\_r^{\min}} \\
\text{Pipe\_Dia}\_r' = \dfrac{\text{Pipe\_Dia}\_r^{\max} - \text{Pipe\_Dia}\_r}{\text{Pipe\_Dia}\_r^{\max} - \text{Pipe\_Dia}\_r^{\min}} \\
\text{Max\_Depth}\_r' = \dfrac{\text{Max\_Depth}\_r - \text{Max\_Depth}\_r^{\min}}{\text{Max\_Depth}\_r^{\max} - \text{Max\_Depth}\_r^{\min}}
\end{cases}
$$

其中，$\text{Elevation}\_r'$和$\text{Drain\_Cap}\_r'$为负向指标（值越大表示该属性越不利于抵御内涝），$\text{Slope}\_r'$、$\text{Pipe\_Dia}\_r'$和$\text{Max\_Depth}\_r'$为正向指标（值越大表示脆弱性越高）。

设定权重向量$\mathbf{w} = \[0.15, 0.15, 0.15, 0.15, 0.4]$，计算各路段与理想解的相对贴近度作为物理脆弱性指数：

**步骤2-加权TOPSIS贴近度**：
$$V\_r = \frac{d\_r^-}{d\_r^+ + d\_r^-} \tag{5-11}$$

其中$d\_r^+ = \sqrt{\sum\_{k=1}^{5} w\_k^2 (x\_{rk}' - 1)^2}$，$d\_r^- = \sqrt{\sum\_{k=1}^{5} w\_k^2 (x\_{rk}' - 0)^2}$。

**5.2.3.3 薄弱环节识别方法**：采用扰动分析法识别系统关键影响因素。当某一指标的权重发生微小扰动时，若系统得分变化剧烈，说明该指标对系统韧性具有显著影响。

**步骤1-基准计算**：以原始权重向量$\mathbf{w} = (w\_1, w\_2, \ldots, w\_9)$计算基准综合得分$sc\_0$。

**步骤2-权重扰动**：依次对第$i$个指标的权重增加10%扰动并归一化：
$$w\_i' = \frac{1.1 w\_i}{1 + 0.1 w\_i} \tag{5-12}$$

**步骤3-敏感度计算**：重新计算综合得分$sc\_i$，计算得分变化率作为敏感度系数：
$$S\_i = \frac{|sc\_i - sc\_0|}{sc\_0 \times 0.1} \tag{5-13}$$

### 5.3 问题三：城市内涝应急疏散路径优化

#### 5.3.1 解题思路与模型选择

本问题旨在建立动态疏散路径优化模型，通过动态时空规划算法（TDSPP）实现总疏散时间最短与路径安全度最高的双目标优化\[5]。该算法将时间维度纳入路径规划，能够根据问题一提供的动态积水数据实时更新路径。相比静态导航仅考虑物理距离最短的局限性，动态规划可根据实时积水深度动态调整路径，有效规避断路风险，从而提升应急疏散效率。

#### 5.3.2 数据处理

1. **图结构构建**：将路网拓扑转换为无向图 $G=(V,E)$，其中节点集 $V$ 包含起点、终点及中间节点{A,B,C,D}，边集 $E$ 包含8条路段，每条边关联长度属性。
2. **时间插值**：通过线性插值实现任意时刻 $t$ 的积水深度估计：
   $$h(road,t) = \sum\_{i=0}^{3} h\_i \cdot \frac{t\_{i+1}-t}{t\_{i+1}-t\_i} \cdot I(t\_i \leq t < t\_{i+1}) \tag{5-14}$$
   其中 $t\_i \in {0,15,30,45,60}$ 为时间节点，$h\_i$ 为对应时刻的积水深度。
3. **状态映射**：根据积水深度映射为通行能力系数 $c(h)$ 和安全度 $s(h)$：
   $$
   (c(h), s(h)) =
   \begin{cases}
   (1.0, 1.0) & h < 5 \\
   (0.8, 0.9) & 5 \leq h < 10 \\
   (0.4, 0.6) & 10 \leq h < 20 \\
   (0.1, 0.2) & 20 \leq h < 30 \\
   (0.0, 0.0) & h \geq 30
   \end{cases}
   $$

#### 5.3.3 模型建立

**5.3.3.1 双目标优化模型**

本问题以疏散时间最短和路径安全度最高为双目标，建立如下优化模型：

$$
\begin{cases}
\min \quad T = \sum\_{(i,j) \in P} \frac{L\_{ij}}{v\_{ij}(t)} \\
\max \quad S = \prod\_{(i,j) \in P} \text{Safety}\_{ij}(t)
\end{cases}
$$

其中：$v\_{ij}(t) = V\_{\text{max}} \cdot c(h\_{ij}(t))$，$V\_{\text{max}} = 500 , \text{m/min}$ 为基础车速。安全度采用乘积形式体现串联风险累积特性。

**5.3.3.2 时空路径成本函数**

采用加权线性和法将双目标转化为单目标优化，构建时空路径成本函数：

$$\text{Cost}(i,j,t) = \alpha \cdot \text{Time}(i,j,t) - \beta \cdot \ln(\text{Safety}(i,j,t)) \tag{5-15}$$

其中 $\alpha = 1.0$，$\beta = 15.0$。负对数形式确保安全度越低惩罚越大，符合风险规避原则。路段旅行时间 $\text{Time}(i,j,t) = \frac{L\_{ij}}{V\_{\text{max}} \cdot c(h)}$，随积水深度动态变化。

#### 5.3.4 模型求解与算法设计

**时变Dijkstra算法步骤**：

**步骤1-初始化**：构建时空网络，每个节点标记为$(node, time)$，初始化优先级队列存储候选路径（包含累计成本、当前节点、当前时间、路径序列、累计耗时、累计安全度），设置起点成本为0。

**步骤2-状态更新**：在每一步迭代中，根据当前时间通过线性插值获取路段积水深度 $h(road,t)$，调用状态映射函数得到通行能力系数 $c(h)$ 和安全度 $s(h)$，若通行能力为0则跳过该路段（断路处理）。

**步骤3-路径搜索**：**步骤3-路径搜索**：鉴于积水深度的时空动态演变，传统静态规划难以保障疏散安全。本文将静态 Dijkstra 算法升级为考虑“水淹阻抗”的**时变 Dijkstra 算法（Time-Dependent Dijkstra）**\[3]。算法实时提取最小成本路径进行动态扩展，并计算路段旅行时间 $\text{Time} = \frac{L}{V\_{\text{max}} \cdot c(h)}$，更新到达下一节点的时间和安全度。引入剪枝策略：若到达同一节点的时间超过已有最优时间+15分钟冗余阈值，则跳过该路径，避免无效搜索。

**步骤4-灵敏度分析**：调整$\beta$值进行参数扫描（$\beta \in {0,10,15,20,30}$），分析不同策略下的路径选择差异，确定最优权衡参数。

基于时变Dijkstra算法计算得到最优疏散路径为**起点→B→D→终点**，总耗时**4.62分钟**，综合安全度**0.3240**。

### 5.4 问题四：城市内涝韧性提升的多目标规划

#### 5.4.1 解题思路

本问题旨在5000万元预算约束下，通过优化管网扩容长度($x\_1$)、调蓄池数量($x\_2$)、透水铺装面积($x\_3$)和预警系统建设($x\_4$)四个决策变量，实现**成本最低、韧性增益最大、退水时间最短**三个目标的协同优化。

本问题具有多目标优化特征，三个目标相互制约：提高韧性需增加工程投入，缩短退水时间需要更大调蓄容积和排水能力。本文采用蒙特卡洛采样结合帕累托支配排序的方法提取帕累托前沿，再通过K-Means聚类获取三类典型方案\[6]。该方法相比NSGA-II更适合混合整数决策空间，计算效率高且结果稳定。

本问题综合问题一积水动态演化与问题二韧性评价结果，以L5等薄弱路段为重点改造对象，集成四种工程措施形成系统性方案。问题二确定的客观权重(排水能力45%、调蓄容积35%、应急响应20%)直接嵌入韧性增益计算，实现前后问题的逻辑闭环。

#### 5.4.2 数据处理

**数据来源**：以附表6给出的四项改造措施的成本、效果与物理边界为输入数据，具体参数设置如下表所示。

| 措施类型  | 参数符号   | 单位 | 取值范围          | 单位成本       |
| :---- | :----- | :- | :------------ | :--------- |
| 管网扩容  | $x\_1$ | m  | $\[0, 5000]$  | 1.8 万元/m   |
| 调蓄池建设 | $x\_2$ | 座  | ${0,1,2,3}$   | 1200 万元/座  |
| 透水铺装  | $x\_3$ | m² | $\[0, 30000]$ | 0.35 万元/m² |
| 预警系统  | $x\_4$ | 套  | ${0,1}$       | 80 万元/套    |

**蒙特卡洛采样**：采用均匀分布对决策变量进行大规模随机采样，生成15,000个初始解。具体采样方案为：$x\_1 \sim U(0,5000)$、$x\_2 \sim U{0,1,2,3}$、$x\_3 \sim U(0,30000)$、$x\_4 \sim U{0,1}$。所有满足预算约束$\text{Cost} \leq 5000$万元的解构成可行解集，用于后续帕累托前沿提取。

**成本计算**：总改造成本由四项措施的线性叠加得到，单位统一为万元：

$$
\text{Cost} = 1.8 x\_1 + 1200 x\_2 + 0.35 x\_3 + 80 x\_4 \tag{5-14}
$$

**韧性增益计算**：韧性增益由排水能力提升、调蓄容积增加、应急响应改善三个维度的归一化增益加权求和得到，权重直接取自问题二的MC-EWM客观权重$(w\_{\text{drain}}, w\_{\text{store}}, w\_{\text{emg}}) = (0.45, 0.35, 0.20)$：

$$
G\_{\text{drain}} = \frac{x\_1}{5000} \times 0.25, \quad
G\_{\text{store}} = \frac{x\_2 \times 800 + x\_3 \times 0.1}{3500}, \quad
G\_{\text{emg}} = x\_4 \times 0.40
$$

$$
\text{Resilience\_Gain} = w\_{\text{drain}} \cdot G\_{\text{drain}} + w\_{\text{store}} \cdot G\_{\text{store}} + w\_{\text{emg}} \cdot G\_{\text{emg}} \tag{5-15}
$$

**退水时间计算**：基于第一问L5路段积水消退的物理机理，将工程改造措施映射为排水速率和调蓄容积的物理增量。基础环境参数设定：初始积水量$V\_0 = 8000 , \text{m}^3$（对应65mm极端降雨的汇水体积），基础排水速率$R\_{\text{base}} = 50 , \text{m}^3/\text{min}$。工程改造带来的增量排水速率与管网扩容长度成正比，增量调蓄容积为调蓄池与透水铺装容量之和：

$$
R\_{\text{extra}} = \frac{x\_1}{5000} \times 0.25 \times 50 = 0.0025 x\_1
$$

$$
V\_{\text{extra}} = 800 x\_2 + 0.1 x\_3
$$

退水时间为剩余积水量除以总排水速率的连续时间解析解：

$$
T\_{\text{drain}} = \frac{\max(0, V\_0 - V\_{\text{extra}})}{\max(1, R\_{\text{base}} + R\_{\text{extra}})} \tag{5-16}
$$

本问题旨在建立多目标优化模型，设计内涝韧性提升综合改造方案，实现成本、韧性、效率的综合最优。采用蒙特卡洛模拟结合帕累托前沿提取的方法，通过K-Means聚类筛选典型方案供决策选择。本问题综合前三个问题的研究成果：问题一提供物理退水机制，问题二提供客观权重体系，问题三验证路径优化效果，最终提出可落地的解决方案。

#### 5.4.2 数据处理

1. **解空间生成**：通过蒙特卡洛模拟生成15000个初始解，决策变量范围为：$x\_1 \in \[0,5000]$ m（管网扩容），$x\_2 \in {0,1,2,3}$（调蓄池数量），$x\_3 \in \[0,30000]$ m²（透水路面），$x\_4 \in {0,1}$（预警系统）。
2. **约束筛选**：施加预算约束 $\text{Cost} \leq 5000$ 万元，筛选有效解。
3. **目标计算**：计算成本、韧性增益和退水时间三个目标值。

#### 5.4.3 模型建立

城市内涝治理与“海绵城市”改造本质上是一个典型的高维多目标优化问题，在实际工程中亟需权衡建设费用、设施效能与防涝韧性\[4]。因此，本文以帕累托最优（Pareto Optimality）理论为指导，建立兼顾总成本最低、韧性增益最大、退水时间最短的多目标优化模型\[4]。

$$
\begin{cases}
\min\limits\_{x \in \Omega} \quad \text{Cost} = 1.8 x\_1 + 1200 x\_2 + 0.35 x\_3 + 80 x\_4 \\
\max\limits\_{x \in \Omega} \quad \text{Resilience\_Gain} = w\_{\text{drain}} \cdot \dfrac{x\_1}{5000} \times 0.25 + w\_{\text{store}} \cdot \dfrac{800 x\_2 + 0.1 x\_3}{3500} + w\_{\text{emg}} \cdot x\_4 \times 0.40 \\
\min\limits\_{x \in \Omega} \quad T\_{\text{drain}} = \dfrac{\max(0, 8000 - 800 x\_2 - 0.1 x\_3)}{\max(1, 50 + 0.0025 x\_1)}
\end{cases}
$$

其中决策变量域$\Omega = {x\_1 \in \[0,5000], x\_2 \in {0,1,2,3}, x\_3 \in \[0,30000], x\_4 \in {0,1}}$。

**约束条件**：

**帕累托最优解定义**：对于可行解$x^a$和$x^b$，若$x^a$在所有目标上都不劣于$x^b$，且至少在一个目标上严格优于$x^b$，则称$x^a$帕累托支配$x^b$。不被任何其他解帕累托支配的解构成帕累托前沿$\mathcal{F}^\*$。

#### 5.4.4 模型求解与算法设计

**求解流程**：采用"蒙特卡洛采样—帕累托筛选—K-Means聚类"三阶段算法。

**阶段一：蒙特卡洛可行解生成**

步骤1：随机生成$N=15000$个初始解$(x\_1^{(i)}, x\_2^{(i)}, x\_3^{(i)}, x\_4^{(i)})$；

步骤2：计算每个解对应的三个目标函数值${\text{Cost}^{(i)}, \text{Resilience\_Gain}^{(i)}, T\_{\text{drain}}^{(i)}}$；

步骤3：筛选满足预算约束$\text{Cost}^{(i)} \leq 5000$的解，保留为候选解集。

**阶段二：帕累托前沿提取**

步骤4：构建目标向量$\mathbf{F}^{(i)} = (\text{Cost}^{(i)}, -\text{Resilience\_Gain}^{(i)}, T\_{\text{drain}}^{(i)})$，其中韧性增益取负号以统一为最小化方向；

步骤5：对于候选解集中的每个解，判断是否存在另一解在所有维度上均严格更优。若存在，则当前解被标记为被支配解；不被任何解支配的解构成帕累托前沿$\mathcal{F}^\*$。

**阶段三：K-Means聚类提取典型方案**

步骤6：将帕累托前沿的三个目标值进行标准化处理（Z-score标准化），消除量纲差异；

步骤7：执行$K=3$的K-Means聚类，将帕累托解集划分为三个簇；

步骤8：计算每个簇的几何中心，将距中心最近的帕累托解确定为中心解，作为该簇的典型代表方案。

**典型方案命名原则**：根据成本从低到高排序，依次命名为"保守防御型（低成本）""均衡拐点型（高性价比）""激进海绵型（最高韧性）"。

**算法参数设置**：蒙特卡洛采样规模$N=15000$，聚类数$K=3$，随机种子seed=42（确保结果可复现）。

**求解步骤**：

**步骤1-解空间生成**：采用蒙特卡洛模拟生成15000个初始解，覆盖所有决策变量组合空间。

**步骤2-约束筛选**：根据预算约束筛选有效解，剔除超出预算的方案。

**步骤3-目标计算**：计算每个有效解的成本、韧性增益和退水时间，其中退水时间基于物理机理推演。

**步骤4-帕累托前沿提取**：通过逐点比较法识别非支配解，构建帕累托最优解集。

**步骤5-典型方案提取**：采用K-Means聚类算法将帕累托解集分为3类，选取距离聚类中心最近的方案作为典型方案。

## 六、模型优缺点

### 6.1 模型优势

（1）**多层级建模，逻辑闭环完整**

本文从积水动态演化（问题一）出发，逐步拓展至韧性评价（问题二）、动态路径优化（问题三）以及工程改造决策（问题四），构建了"物理机理—系统评价—路径决策—优化设计"的完整分析链条。各部分之间数据共享、结果联动，实现了从状态刻画到决策输出的闭环建模。

（2）**物理机制与数学模型高度耦合**

在积水演化建模中，引入汇水面积修正与管网顶托失效机制，通过分段函数刻画排水能力突变，使模型能够反映低洼区域在极端降雨下的非线性失效特征。同时结合参数反演，使模型结果与观测数据保持较高一致性，增强了物理解释力。

（3）**评价体系客观性强，诊断能力突出**

在韧性评价中，通过蒙特卡洛模拟构造样本空间，结合熵权法进行客观赋权，有效避免了单一观测数据下权重不可计算的问题。进一步引入敏感性分析，不仅获得综合评分，还能够识别关键薄弱指标与高风险路段，实现"定量评价+结构诊断"的统一。

（4）**动态决策模型具备时空预见性**

针对传统最短路径方法的局限性，构建了基于时变边权的路径优化模型，将"时间"作为状态变量引入搜索过程，并通过对安全度的对数变换实现多目标标量化。该方法能够预判未来路段失效风险，有效避免"途中断路"问题，提高疏散策略的可靠性。

（5）**多目标优化与工程决策有效结合**

在改造方案设计中，将工程措施与韧性指标建立定量映射关系，构建多目标混合整数优化模型，并利用NSGA-II算法获得帕累托最优解集。进一步通过聚类提取典型方案，实现从"解集"到"可实施方案"的转化，具有较强工程指导意义。

### 6.2 模型不足

（1）**路段水力独立假设限制系统联动性**

在积水演化过程中，将各路段视为相互独立的汇水单元，未显式刻画路段之间的地表径流传递与管网耦合关系，可能低估局部区域的汇水叠加效应。

（2）**优化模型中效用函数存在线性近似**

在第四问中，韧性提升与退水效率均采用线性表达形式，未完全刻画实际工程措施之间可能存在的非线性耦合关系，可能对高投入方案的效果估计存在偏差。

（3）**参数标定依赖有限观测数据**

模型参数主要通过少量观测点进行反演得到，在数据规模较小的情况下，可能存在一定的不确定性，对极端场景的外推能力仍有提升空间。

### 6.3 模型改进方向

（1）**引入空间降雨场与分布式水文模型**

未来可结合雷达降雨数据或格点降雨模型，构建空间非均匀降雨场，并引入分布式水文模型（如栅格水动力模型），以提高积水模拟的空间精度。

（2）**构建路网水力耦合模型**

在现有基础上引入路段间径流交换机制或简化管网拓扑结构，建立路网级联动模型，从而更真实地反映积水传播与汇集过程。

（3）**融合交通流与行为决策模型**

在动态路径优化中，可引入交通流模型（如元胞自动机或宏观流模型），并考虑个体决策差异与信息延迟，从而提升疏散路径的现实适用性。

（4）**构建非线性或机器学习增强模型**

针对工程措施的综合效应，可采用非线性回归或机器学习方法（如随机森林、神经网络）建立更精细的"措施—效果"映射关系，以提高优化模型的拟合能力。

def simulate_water_depth(params, df_roads, rainfall_rate_m_min):
    # 物理参数：Catchment_Factor(街区面积倍数), Cd(管网老化折减系数)
    Catchment_Factor, Cd = params
    n_roads = len(df_roads)
    n_time = len(rainfall_rate_m_min)

    h_m = np.zeros(n_roads)
    h_cm_history = np.zeros((n_time, n_roads))
    dt = 1.0

    base_area = df_roads['长度(m)'] * df_roads['路宽(m)']
    
    # 物理机制1：基于坡度梯度的街区真实汇水效应放大
    catchment_area = base_area.values * (1.0 + Catchment_Factor * df_roads['坡度(‰)'].values / 10.0)
    
    psi = df_roads['径流系数'].values
    drain_capacity = df_roads['排水能力(m³/min)'].values
    is_cement = (df_roads['路面类型'] == '水泥').values
    Z_surf = df_roads['最低标高(m)'].values
    
    # 硬性锁死物理常识：水泥地的下渗率极低 (约 3mm/h)
    I_cement = 0.00005 

    for t in range(n_time):
        # 推理公式法 Q = psi * q * F
        R_volume = rainfall_rate_m_min[t] * catchment_area * psi * dt
        infiltration_volume = np.where(is_cement, I_cement * base_area.values * dt, 0.0)
        
        current_water_volume = h_m * base_area.values
        
        # 物理机制2：引入地下管网的“顶托效应”(Backwater Effect)
        backwater_factor = np.maximum(0.1, 1.0 - (h_m / 0.4))
        dynamic_drain = drain_capacity * Cd * backwater_factor * dt 
        
        Q_volume = np.minimum(dynamic_drain, current_water_volume + R_volume)
        new_water_volume = current_water_volume + R_volume - Q_volume - infiltration_volume
        
        h_m = np.maximum(0, new_water_volume / base_area.values)
        
        # 物理机制3：城市低洼漫溢模型
        for i in range(n_roads):
            if h_m[i] > 0.15: # 积水越过15cm路缘石
                overflow_vol = (h_m[i] - 0.15) * base_area.values[i] * 0.2 
                lowest_idx = np.argmin(Z_surf)
                
                # 水往低处流
                if Z_surf[lowest_idx] < Z_surf[i]:
                    h_m[i] -= overflow_vol / base_area.values[i]
                    h_m[lowest_idx] += overflow_vol / base_area.values[lowest_idx]
                    
        h_cm_history[t, :] = h_m * 100.0

    return h_cm_history

def objective_function(params, df_roads, rainfall_rate_m_min, obs_data):
    h_cm_history = simulate_water_depth(params, df_roads, rainfall_rate_m_min)
    simulated, observed = [], []
    for t in [15, 30, 45, 60]:
        simulated.extend(h_cm_history[t, :])
        observed.extend(obs_data[t])
    
    rmse = np.sqrt(np.mean((np.array(simulated) - np.array(observed))**2))
    return rmse

def map_rules(h):

    return capacity, safety

def get_truncated_normal(mean, low, high, sd):
    return truncnorm((low - mean) / (sd + 1e-7), (high - mean) / (sd + 1e-7), 
                     loc=mean, scale=sd)

def run_system_topsis(df, n_samples=500):
    np.random.seed(2026)
    sim_matrix = np.zeros((n_samples, len(df)))
    for i in range(len(df)):
        low, high = min(df['Worst'][i], df['Ideal'][i]), max(df['Worst'][i], df['Ideal'][i])
        gen = get_truncated_normal(df['Actual'][i], low, high, abs(high-low)*0.15)
        sim_matrix[:, i] = gen.rvs(n_samples)

    # 极向标准化
    norm_sim = np.zeros_like(sim_matrix)
    for i in range(len(df)):
        v = sim_matrix[:, i]
        if df['Direction'][i] == '+':
            norm_sim[:, i] = (v - df['Worst'][i]) / (df['Ideal'][i] - df['Worst'][i] + 1e-9)
        else:
            norm_sim[:, i] = (df['Worst'][i] - v) / (df['Worst'][i] - df['Ideal'][i] + 1e-9)

    # 熵权法（数值稳定版本）
    norm_sim_safe = np.clip(norm_sim, 1e-10, 1)
    col_sums = np.sum(norm_sim_safe, axis=0) + 1e-10
    p = norm_sim_safe / col_sums
    e = - (1 / np.log(n_samples)) * np.sum(p * np.log(p), axis=0)
    weights = (1 - e) / np.sum(1 - e)

    # TOPSIS 现状打分
    norm_act = np.array([(df['Actual'][i]-df['Worst'][i])/(df['Ideal'][i]-df['Worst'][i]) 
                         if df['Direction'][i]=='+'
                         else (df['Worst'][i]-df['Actual'][i])/(df['Worst'][i]-df['Ideal'][i]) 
                         for i in range(len(df))])
    z_act = norm_act * weights
    d_pos = np.sqrt(np.sum((z_act - 1.0*weights)**2))
    d_neg = np.sqrt(np.sum((z_act - 0.0*weights)**2))
    return d_neg / (d_pos + d_neg), weights, norm_act

sys_score, sys_weights, sys_norm_act = run_system_topsis(df_idx)

import heapq
import math
import numpy as np
import networkx as nx
import matplotlib.pyplot as plt
import os

# 全局参数配置
BETA_DEFAULT = 15.0      # 安全惩罚系数
ALPHA_DEFAULT = 1.0      # 时间权重
PRUNING_BUFFER = 15.0    # 剪枝时间冗余(min)
V_MAX = 500.0            # 基础车速 500 m/min (30 km/h)
START_TIME = 36.0        # 核心剧本触发点

plt.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'Microsoft YaHei']
plt.rcParams['axes.unicode_minus'] = False
plt.rcParams['figure.dpi'] = 300

IMG_PREFIX = "answer3图"

# 附表1：路段长度
L = {'L1': 450, 'L2': 320, 'L3': 580, 'L4': 260,
     'L5': 390, 'L6': 410, 'L7': 290, 'L8': 520}

# 附表4：时序水深数据
water_depth = {
    'L1': [0, 8.0, 14.0, 18.0, 16.0], 'L2': [0, 12.0, 18.0, 24.0, 20.0],
    'L3': [0, 5.0, 9.0, 12.0, 10.0],  'L4': [0, 7.0, 13.0, 17.0, 15.0],
    'L5': [0, 18.0, 28.0, 32.0, 29.0], 'L6': [0, 6.0, 11.0, 15.0, 13.0],
    'L7': [0, 4.0, 7.0, 9.0, 8.0],    'L8': [0, 10.0, 16.0, 22.0, 19.0]
}
T_points = [0, 15, 30, 45, 60]

# 拓扑定义
edges = [
    ('起点', 'A', 'L1'), ('A', '终点', 'L5'),   
    ('起点', 'B', 'L4'), ('B', 'D', 'L7'), ('D', '终点', 'L8'),
    ('A', 'C', 'L2'), ('C', '终点', 'L6')
]

graph = {}
for u, v, road in edges:
    graph.setdefault(u, []).append((v, road))
    graph.setdefault(v, []).append((u, road))

def get_h(road_id, t):
    """根据时间 t 线性插值获取当前积水深度"""
    if t >= 60: return water_depth[road_id][-1]
    return np.interp(t, T_points, water_depth[road_id])

def get_capacity_and_safety(h):
    """附表3规则映射"""
    if h < 5: return 1.0, 1.0
    elif 5 <= h < 10: return 0.8, 0.9
    elif 10 <= h < 20: return 0.4, 0.6
    elif 20 <= h < 30: return 0.1, 0.2
    else: return 0.0, 0.0

def time_varying_dijkstra(start, end, start_t, alpha=ALPHA_DEFAULT, beta=BETA_DEFAULT):
    """时变 Dijkstra 算法 (TDSPP)"""
    pq = [(0.0, start, start_t, [start], 0.0, 1.0)]
    best_cost = {start: 0.0}

    while pq:
        cost, u, curr_t, path, duration, curr_saf = heapq.heappop(pq)
        if u == end: return path, duration, curr_saf
            
        for v, road_id in graph.get(u, []):
            h = get_h(road_id, curr_t)
            c, s = get_capacity_and_safety(h)
            if c == 0 or s == 0: continue
            
            edge_t = L[road_id] / (V_MAX * c)
            new_t = curr_t + edge_t
            
            if v in best_cost and new_t > best_cost.get(v, 0) + PRUNING_BUFFER: continue
            
            new_saf = curr_saf * s
            edge_cost = alpha * edge_t - beta * math.log(s)
            new_cost = cost + edge_cost
            
            if v not in best_cost or new_cost < best_cost[v]:
                best_cost[v] = new_cost
                heapq.heappush(pq, (new_cost, v, new_t, path + [v], duration + edge_t, new_saf))
                
    return None, float('inf'), 0.0
